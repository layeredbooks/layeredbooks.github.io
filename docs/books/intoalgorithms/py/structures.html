5.1 Disjoint sets

Consider \(n\) numbers, from \(0\) up to and including \(n-1\).

Let these numbers define \(n\) individual sets, with each of the sets having one member.

We might then want to join two of these sets.

For example, joining the set \(i\) with the set \(j\) gives a new set, consisting of the union of the two sets.

Given a number \(a\), we might want to ask to which set this number belongs?

We can accomplish this by assigning, for each set, a representative of the set.

Initially, all numbers represent their own sets.

After a join, however, we need to select a representative for the resulting set, consisting of the joined sets.

We form trees, with directed links going towards the root of the tree.

5.2 Fenwick trees

We describe Fenwick trees. Inspiration is mostly taken from Laaksonen (2020).

We start with an integer number \(v\).

We are interested in the bits that are set to 1 in the binary representation of \(v\).

Let’s assign indices to the bits, starting with 0 for the rightmost bit.

As an example, the number 52, which is written in binary as \[ 0110100 \] has bits 2, 4, and 5 set to 1.

Using the notations \(b_1 = 2\), \(b_2 = 4\), and \(b_3 = 5\), we can write the number 52 as \[ 2^{b_1} + 2^{b_2} + 2^{b_3} = 2^2 + 2^4 + 2^5 = 4 + 16 + 32 = 52 \]

Generalizing, we can consider a number \(v\) with bits \[ b_1, b_2, \ldots, b_k \] set to 1. This number can be expressed as \[ v = \sum_{i=1}^k 2^{b_i} \tag{5.1}\]

Given a number \(v\), define the function \(p(v)\) as a function that returns the largest power of two that divdes \(v\).

From Equation 5.1, we see that \[ p(v) = 2^{b_1} \]

Suppose now that we subtract \(p(v)\), resulting in a new number \(v_1 = v - p(v)\). We can note that \(p(v_1) = 2^{b_2}\).

This means that we can compute the terms in Equation 5.1 by iteratively computing the result of the function \(p\) and then subtracting the result.

Noting that \(p(1) = 0\), we can do this as long as our number is greater or equal to 1.

We can write this as an algorithm, as

\begin{algorithm} \caption{Iterative usage of the function p} \begin{algorithmic} \Procedure{p-iter}{$v$} \While{$v$ $\ge$ 1} \State $v = v - p(v)$ \EndWhile \EndProcedure \end{algorithmic} \end{algorithm}

As an example, running the algorithm Algorithm 5.1 on the number 52 gives, for the values \(p(v)\), the sequence 4, 16, 32.

We now consider the problem of summing values in an array. We assume that the array-indexing starts at the value 1.

Suppse we want to sum the indices from 1 up to and including 52. Noting that 52 = 4 + 16 + 32, we could make this summation by first summing from from 1 up to and including 32, then from 33 up to and including 48, and finally from 49 up to and including 52.

Given an array \(a\), we could create another array \(t\), where at index \(v\) in \(t\), we store the sum \[ t_v = \sum_{i = v - p(v) + 1}^v a_i \tag{5.2}\]

We refer to the array \(t\) as a Fenwick tree.

For our example at hand, using the number 52, we would compute the sum from 1 to 52 as \[ \sum_{i = 1}^{52} a_i = \sum_{i = 49}^{52} a_i + \sum_{i = 33}^{48} a_i + \sum_{i = 1}^{32} a_i \] Using Equation 5.2, we can write this as \[ \sum_{i = 1}^{52} a_i = t_{52} + t_{48} + t_{32} \]

We can modify Algorithm 5.1 so that it computes a sum, using the array \(t\). This results in

\begin{algorithm} \caption{Computing the sum in a Fenwick tree} \begin{algorithmic} \Procedure{p-sum}{$v$, $t$} \State $sum = 0$ \While{$v$ $\ge$ 1} \State $sum = sum + t_v$ \State $v = v - p(v)$ \EndWhile \EndProcedure \end{algorithmic} \end{algorithm}

Given an array \(a\), we can compute range sums using Algorithm 5.2, using a Fenwick tree \(t\).

We might wonder how to update \(t\), if the array \(a\) is updated?

We see, from Equation 5.2, which elements \(a_i\) that are involved in a given sum. So if an array value at index \(i\) is updated, we must update \(t\) at all places where the term at index \(i\) is included in a sum.

Looking at Algorithm 5.2, we see that when computing a sum, we visit the places \(t_v\), with v being updated as \[ v = v - p(v) \] and ending as soon as \(v\) becomes 1 or smaller than 1.

So if we update index \(i\) in \(a\), we could update \(t\) for index \(i\) as well as all indices above \(i\), computed as \[ i = i + p(v) \]

This gives an algorithm for update, as

\begin{algorithm} \caption{Updating a Fenwick tree} \begin{algorithmic} \Procedure{p-update}{$v$, $\delta$, $t$} \State $v_{max}$ = $length(t)$ \While{$v$ $\le$ $v_{max}$} \State $t_v = t_v + \delta$ \State $v = v + p(v)$ \EndWhile \EndProcedure \end{algorithmic} \end{algorithm}
